# 经典题目

## 题目一：归因

数据如下，用户从渠道一进入后，后面渠道号为空，要求填补对应的渠道号

```
userid channel_id event_time 
u001   dy     1577862283000
u001   null   1577862283001
u001   null   1577862283002
u001   null   1577862283003
u001   ks     1577862284000
u001   null   1577862284001
u001   null   1577862284002
u001   null   1577862284003
```

结果如下:

```
userid channel_id event_time 
u001   dy     1577862283000
u001   dy   	1577862283001
u001   dy   	1577862283002
u001   dy   	1577862283003
u001   ks     1577862284000
u001   ks   	1577862284001
u001   ks   	1577862284002
u001   ks   	1577862284003
```

解析:其实就是一个逻辑上的分组，虽然都是u001一个用户，但是只要能保证dy后ks前的一个分组，ks的一个分组，做完正确的分组，取一下渠道号即可。重点就在于逻辑上分组的实现。

```sql
分组： count(null) over(partition by userid order by event_time desc) as lg_group
取渠道号： max(channel_id) over(partition by userid,lg_group) 
```

同样的 原理 也适用于用户会话的串联，例如从未成单开始的会话日志 一直到成单的日志，可以对成单之前的日志进行填补订单号。

##  题目二:  最大连续登陆天数

有一个登陆记录表，求每个用户的最大连续登陆天数

```
userid  login_time
u001 		2022-01-01
u002 		2022-01-01
u001 		2022-01-02
u002 		2022-01-02
u001 		2022-01-03
u002 		2022-01-04
u001 		2022-01-05
u002 		2022-01-05
u001 		2022-01-06
u002 		2022-01-06
u003 		2022-01-06
u003 		2022-01-07
u003 		2022-01-08
```

结果如下:

```
userid  max(login)
u001 		3
u002 		3
u003    3
```

解析: 还是 逻辑分组的思路，想一下如果能把用户连续登陆的日期段划分一组，那么只需要看一下 哪个分组的count值最大 即为所求。

```sql
分组: 
	先标记顺序:row_number() over(partition by userid,order by login_time) rnk
	做差可得逻辑分组的标识: date_add(login_time ,-rnk) as lg_group (同一用户连续登陆的时间段为同一个		日期)
用户最大的分组count: 
	select user_id,max(cnt)
	from (
     select user_id,lg_group,count(0) cnt from tmp group by user_id,lg_group
  ) t
  group by user_id;
```
