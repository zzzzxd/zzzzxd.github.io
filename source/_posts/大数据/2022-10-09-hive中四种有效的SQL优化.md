### 数据准备

首先虚拟一张表user，字段为：

（1）**user_id**（用户的id）；

（2）**age**（用户的年龄）；

（3）**sex**（用户的性别）；

（4）**order_id**（用户的订单编号，一个用户可能有多个订单编号）。

另设一张表，叫order，里面包含order_id, order_date两个字段。

### multi-table-insert优化union all

**原SQL**（目标是往一个新表user_new里，插入最大值和最小值的数据）

```sql
insert into table user_new
select sex, max(age) as stat, 'max' as class
from user
group by sex
union all
select sex, min(age) as stat, 'min' as class
from user
group by sex;
```

**优化SQL**（完成相同的目标，使用multi-table-insert写法）

```sql
from user
insert into table user_new
select sex, max(age) as stat, 'max' as class
group by sex
insert into table user_new
select sex, min(age) as stat, 'min' as class
group by sex;
```

如果SQL是在MapReduce上运行的话，进行这种改写可以减少中间结果的产生，即降低了读写（I/O）资源的消耗，提升程序的效率和速度。具体原理是减少了MapReduce作业数量。

### 使用grouping sets替代union all

**原SQL**（目标是进行2个group by，但后一个的分组字段是前者的子集）

```sql
select * from (
  select sex, age, count(1) as user_num
  from user
  group by sex, age
  union all
  select sex, null as age, count(1) as user_num
  from user
  group by sex) a;
```

**优化SQL**（完成相同的目标，使用grouping sets写法）

```sql
select sex, age, count(1)
from user
group by sex, age
grouping sets((sex),(sex,age));
```

### **使用子查询对count(distinct)可能造成的数据倾斜进行优化**

**原SQL**（统计每种性别有多少种不同年龄

```sql
select sex, count(distinct age) as age_num
from sex
group by sex;
```

**优化SQL**（完成相同的目标，使用子查询避免数据出现倾斜而导致性能问题）

```sql
select sex, count(1) as age_num
from(
  select sex, age, count(1) as num
  group by sex, age
) a;
```

### 使用left join替代not in完成取A表中没有但B表中有的数据

**原SQL**（查找订单日期不在2020-10-23日的用户的user_id）

```sql
select user_id
from user
where order_id not in(
  select order_id from order
  where order_date != '2020-10-23'
); 
```

**优化SQL**（完成相同的目标，用left join去实现，避免not in不走索引带来的问题）

```sql
select a.user_id
from user a
left join
order b
on a.order_id = b.order_id and b.order_date != '2020-10-23'
where b.user_id is null;
```
