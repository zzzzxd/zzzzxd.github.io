## 前言

linux系统中充满着各种各式的符号，尤其是在书写shell脚本时候，更是明显，今天索性把所有常见的符号进行总结，方便记忆。

## 引号

| 符号             | 含义                                                         |
| ---------------- | ------------------------------------------------------------ |
| 反引号:`` 或 $() | 优先执行,先执行里面的命令. 一般创建/打包的时候给文件名加上个日期. |
| 单引号: ''       | 所见即所得,单引号里面的内容会原封不动的输出.                 |
| 双引号: ""       | 与单引号类似,但是对引号的里面特殊符号会进行解析(运行)        |
| 不加引号:        | 与双引号类似,同时支持{}.                                     |

```shell
#case01:单引号,所见及所得
[root@linx01 ~]# echo 'root $UID `whoami` {1..5}'
root $UID `whoami` {1..5}
#case02:双引号 ，会进行解析
[root@linx01 ~]# echo "root $UID `whoami` {1..5}"
root 0 root {1..5}
#case03:反引号 运行指令的
[root@linx01 ~]# echo "root $UID whoami {1..5}"
root 0 whoami {1..5}
```

## 重定向符号

| 重定向符号       | 含义                                                   |
| ---------------- | ------------------------------------------------------ |
| >或 1>           | 标准输出重定向,先清空文件内容,然后写入.                |
| >>或 1>>         | 标准输出追加重定向, 把内容追加到文件末尾.              |
| 2>               | 标准错误输出重定向, 先清空文件,然后写入错误信息到文件. |
| 2>>              | 标准错误追加输出重定向, ,把错误信息追加到文件末尾.     |
| > test.log 2>&1  | 把标准错误输出与标准输出合并,然后重定向到test.log中.   |
| >> test.log 2>&1 | 追加到test.log中.                                      |
| &>test.txt       | 正确输出和错误输出重定向到test.txt中                   |
| __&>>test.txt__  | 正确输出和错误输出 追加到test.txt中                    |
| <或0<            | 标准输入重定向                                         |
| <<或0<<          | 标准输入追加重定向                                     |

补充：0,1,2都是文件打开的描述符 每个程序都自带这3个打开描述符号。

以上都比较简单，就不给事例了。

## 通配符

| 通配符 | 含义                                                       |
| ------ | ---------------------------------------------------------- |
| *      | 匹配任意0个或多个字符 字符串                               |
| ?      | __匹配任意1个字符，有且只有一个字符  __ 千万不要和正则混了 |
| {}     | 生成序列                                                   |
| ?      | 任意一个字符                                               |
| []     | 其中一个即可                                               |
| [^]    | 排除其中的                                                 |
| \|     | 管道符号                                                   |
| ～     | 用户的家目录，超级用户为 /root， 普通用户为 /用户名        |
| -      | 代表上一次（相对于当前路径） 用户所在的路径                |
| .      | 代表当前目录(正则中的任意字符)                             |
| ..     | 代表上一级目录                                             |

我们对其中一些字符进行演示

### {}、*

```shell
# * 匹配所有文件
[root@linx01 dev]# rm -rf *    
# {} 生成序列
[root@linx01 dev]# touch test{1..10}
[root@linx01 dev]# ls
test1  test10  test2  test3  test4  test5  test6  test7  test8  test9
# 利用*匹配特定的文件
[root@linx01 dev]# ls test1*
test1  test10
```

### []、[^]

```shell
#利用 {}生成文件
[root@linx01 dev]# touch {a..z}.sh
[root@linx01 dev]# ls
a.sh  c.sh  e.sh  g.sh  i.sh  k.sh  m.sh  o.sh  q.sh  s.sh  u.sh  w.sh  y.sh
b.sh  d.sh  f.sh  h.sh  j.sh  l.sh  n.sh  p.sh  r.sh  t.sh  v.sh  x.sh  z.sh
#利用[] 匹配开头是a-c其中，以sh结尾的文件
[root@linx01 dev]# ls [a-c].sh
a.sh  b.sh  c.sh
#利用[^] 匹配开头不是a-c其中并且以sh结尾的文件
[root@linx01 dev]# ls [^a-y].sh
z.sh
```

### -

```shell
[root@linx01 dev]# pwd
/opt/dev
[root@linx01 dev]# cd /etc/sysconfig/network-scripts/
[root@linx01 network-scripts]# pwd
/etc/sysconfig/network-scripts
# 利用- 跳回上次所在路径
[root@linx01 network-scripts]# cd -
/opt/dev
[root@linx01 dev]# pwd
/opt/dev
```

### 综合练习

查找 /etc 目录下包含 hosts 字符串的所有文件

```shell
[root@linx01 dev]# find /etc/ -type f -name "*host.*"
/etc/host.conf
```

查找 /etc 目录下的所有文件，规则是文件名的第一个字符为任意字符，第二个字符为o、p、q中的任意一个字符，第三个字符不能在a到r之间，结尾是ts。

```shell
[root@linx01 dev]# find /etc/ -type f -name "?[opq][^a-r]ts"
/etc/hosts

```
