### **du和df的区别：**

``` shell
df命令是从文件系统考虑的，不光要考虑文件占用的空间，还要统计被命令或程序占用的空间(最常见的就是文件已经删除，但是程序并没有释放空间)。
du命令是面向文件的，只会计算文件或目录占用的
```

### **查看磁盘整体情况**

#### 查看整体挂载情况

```shell
df -h              
```

![WeChat069a76a447ca86aad5fec1408ef2fbde](1.png)

#### 查看指定目录

  ```shell
  df -h /opt/data/
  ```

![WeChatfddb958643be29daa7380f9e2a7f2ab3](2.png)

### **查看文件夹的占用情况**

#### 查看当前目录每个文件夹的情况。

```shell
du --max-depth=1 -h              
```

![WeChatfddb958643be29daa7380f9e2a7f2ab3](3.png)

#### 指定目录

```shell
du --max-depth=1 -h  /opt/data/              
```

![WeChat0e80cc2a684da5f972f3d507563465d2](4.png)

#### 计算文件夹大小

(**为了快算显示，同时也只是想查看目录整体占用大小。可以直接使用du -sh 命令，如果想查看指定目录，直接在命令后根目录即可**)

```shell
du -sh /usr/              
```

![WeChat0dadf840b9fa7087478e4c7700ef1761](5.png)

### 小结

其中df -h和du -sh使用的比较多，一个统计整体磁盘情况，一个看单独目录点用情况，而命令du --max-depth=1 -h查看了目录下文件夹占用情况，使用比较少，可以用du -sh代替，而且命令较长，当然并不是说它没用。
