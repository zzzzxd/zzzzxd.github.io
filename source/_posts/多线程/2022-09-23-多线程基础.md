## 线程状态

线程的状态，是重点中的重点，在生产中遇到问题的时候，就需要观察线程状态来排查问题，比如用jstack查看堆栈信息时候，就需要重点关注block状态的线程。

### 状态的解释

- new: 线程刚刚创建，还没有启动
- Runnable：可运行状态，可由线程调度器安排执行，具体分为：Ready等待调度，Running 正在执行
- Waiting：等待被唤醒
- Timed Waiting：隔一段时间，自动唤醒
- Blocked：正在等锁
- Terminated：线程结束

### lock和synchronized的区别

​	lock是用的juc的锁，基于cas实现，cas都是忙等，不是阻塞而是waiting，而synchronized是等锁，blocked，但两者都是阻塞态。

## 线程启动

介绍几种常见的线程启动的方式

- hread和Runnable 两种

  ```java
  new MyThread().start();
  new Thread(new MyRun()).start();
  ```

- ThreadPool

  ```java
  ExecutorService service = Executors.newCachedThreadPool();
  service.execute(() -> {
      System.out.println("Hello ThreadPool");
  });
  ```

- ThreadPool和Callable

  ```java
  ExecutorService service = Executors.newCachedThreadPool();
  Future<String> f = service.submit(new MyCall());
  String s = f.get();
  ```

- Thread和FutureTask

  ```java
  FutureTask<String> task = new FutureTask<>(new MyCall());
  Thread t = new Thread(task);
  t.start();
  System.out.println(task.get());
  ```

## 线程打断

### 有关线程打断的三个方法

- interrupt方法:名为打断，但是一定注意，其本质是设置线程内部的中断标记位，发现标记修改后做什么样的反应，是线程自己实现
- isInterrupt方法:查询标记位
- interrupted方法:查询当前标记位是否被打断过，如果打断过，重置标记位

## 线程结束

### 优雅结束线程的几种方式

#### 利用volatile变量设置标志位通知线程结束

```java
private static volatile boolean running = true;

public static void main(String[] args) {
    Thread t = new Thread(() -> {
        long i = 0L;
        while (running) {
            //wait recv accept
            i++;
        }
        System.out.println("end and i = " + i); //4168806262 4163032200
    });

    t.start();
    SleepHelper.sleepSeconds(1);
    running = false;
}
```

#### 利用interupt内部标记

```java
public static void main(String[] args) {
    Thread t = new Thread(() -> {
        while (!Thread.interrupted()) {
            //sleep wait
        }
        System.out.println("t1 end!");
    });
    t.start();
    SleepHelper.sleepSeconds(1);
    t.interrupt();
}
```

#### 利用park方法配合interupt

```java
public static void main(String[] args) {
   Thread t = new Thread(() -> {
        System.out.println("1");
        LockSupport.park();
        System.out.println("2");
    });

    t.start();
    SleepHelper.sleepSeconds(1);
    t.interrupt();
}
```
