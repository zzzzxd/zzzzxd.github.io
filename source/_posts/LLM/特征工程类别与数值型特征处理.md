# **特征工程 数值 + 类别特征处理**

这是 **机器学习项目中最常见的预处理模式**，尤其适用于：

- 结构化数据（如 CSV、数据库表）
- 混合了数值和文本/类别特征
- 使用线性模型、SVM、神经网络等



## **🎯 为什么这么做？核心原因**

| 问题                           | 解决方案                            |
| ------------------------------ | ----------------------------------- |
| 数值特征和类别特征处理方式不同 | 不能用同一套规则                    |
| 缺失值处理策略不同             | 数值用均值/中位数，类别用“缺失”标签 |
| 缩放需求不同                   | 数值需要标准化，类别不需要          |
| 编码方式不同                   | 类别需要转为数值（如 One-Hot）      |

✅ 所以：**必须分类型处理，再组合**

------



## **🔧 标准五步流程（你的代码结构完美匹配）**

### **✅ 第一步：自动识别特征类型**

```python
numerical_features = X.select_dtypes(exclude=['object']).columns
categorical_features = X.select_dtypes(include=['object']).columns
```

- **exclude=['object']**：所有非文本列（int, float）→ 数值型
- **include=['object']**：所有文本列 → 类别型

📌 也可用：

```python
X.select_dtypes(include=['number'])  # 更明确
X.select_dtypes(include=['category'])  # 类别类型
```

### **✅ 第二步：为数值特征定义处理流水线**

```python
numerical_transformer = Pipeline(steps=[
    ('fillNA', SimpleImputer(strategy='mean')),     # 填补缺失值
    ('scaler', StandardScaler())                   # 标准化
])
```

#### **常见策略：**

| 操作     | 可选参数                                      |
| -------- | --------------------------------------------- |
| 填补缺失 | **'mean'**, **'median'**, **'most_frequent'** |
| 标准化   | **StandardScaler**（Z-score）                 |
| 归一化   | **MinMaxScaler**（0~1）                       |

📌 树模型（如 Random Forest、XGBoost）**不需要标准化**，可省略。

### **✅ 第三步：为类别特征定义处理流水线**

```python
categorical_transformer = Pipeline(steps=[
    ('fillNA', SimpleImputer(strategy='constant', fill_value='NaN')),
    ('onehot', OneHotEncoder(handle_unknown='ignore'))
])
```

#### **常见策略：**

| 操作                        | 说明                                                  |
| --------------------------- | ----------------------------------------------------- |
| 填补缺失                    | 用 **'constant'** 策略，填 **'Missing'** 或 **'NaN'** |
| 编码方式                    | **OneHotEncoder**, **OrdinalEncoder**                 |
| **handle_unknown='ignore'** | 测试集出现新类别时不报错                              |

📌 高基数类别（如 用户ID）不适合 One-Hot，应改用 Target Encoding 或 Embedding。

### **✅ 第四步：组合所有处理规则**

```python
preprocessor = ColumnTransformer(
    transformers=[
        ('num', numerical_transformer, numerical_features),
        ('cat', categorical_transformer, categorical_features)
    ],
    remainder='drop'  # 可选：丢弃未指定列
)
```

- **ColumnTransformer** 是关键：**按列分组处理，自动拼接结果**
- 支持稀疏矩阵，高效处理 One-Hot 膨胀

------



### **✅ 第五步：应用预处理器，并转为 DataFrame（可选）**

```python
x_train = pd.DataFrame(
    preprocessor.fit_transform(x_train).toarray(),
    columns=preprocessor.get_feature_names_out()
)

x_test = pd.DataFrame(
    preprocessor.transform(x_test).toarray(),
    columns=preprocessor.get_feature_names_out()
)
```

#### **说明：**

- **.fit_transform(train)**：在训练集上学习并转换
- **.transform(test)**：用训练集的规则转换测试集（避免数据泄露）
- **.toarray()**：稀疏矩阵 → 稠密矩阵（⚠️ 仅当内存允许）
- **get_feature_names_out()**：获取新特征名，如 **num__age**, **cat__color_red**

📌 **注意**：如果特征太多，**.toarray()** 可能导致内存爆炸！

------



## **📦 完整代码模板（推荐收藏）**



```python
from sklearn.compose import ColumnTransformer
from sklearn.pipeline import Pipeline
from sklearn.impute import SimpleImputer
from sklearn.preprocessing import StandardScaler, OneHotEncoder
import pandas as pd

# 1. 分离特征类型
numerical_features = X.select_dtypes(include=['number']).columns
categorical_features = X.select_dtypes(include=['object']).columns

# 2. 数值特征处理
numerical_transformer = Pipeline(steps=[
    ('imputer', SimpleImputer(strategy='median')),  # 用中位数更鲁棒
    ('scaler', StandardScaler())
])

# 3. 类别特征处理
categorical_transformer = Pipeline(steps=[
    ('imputer', SimpleImputer(strategy='constant', fill_value='Missing')),
    ('onehot', OneHotEncoder(drop='first', handle_unknown='ignore'))  # drop='first' 防多重共线性
])

# 4. 组合处理器
preprocessor = ColumnTransformer(transformers=[
    ('num', numerical_transformer, numerical_features),
    ('cat', categorical_transformer, categorical_features)
])

# 5. 应用处理
X_train_processed = preprocessor.fit_transform(X_train)
X_test_processed = preprocessor.transform(X_test)

# 6. （可选）转为 DataFrame
feature_names = preprocessor.get_feature_names_out()
X_train_df = pd.DataFrame(X_train_processed.toarray(), columns=feature_names)
```

## **✅ 最佳实践建议**

| 建议                                             | 说明                                     |
| ------------------------------------------------ | ---------------------------------------- |
| ✅ 使用 **Pipeline** + **ColumnTransformer**      | 避免数据泄露，代码整洁                   |
| ✅ 训练集 **fit_transform**，测试集 **transform** | 保证一致性                               |
| ✅ 用 **get_feature_names_out()** 获取特征名      | 方便调试和解释                           |
| ⚠️ 谨慎使用 **.toarray()**                        | 高基数类别会导致内存爆炸                 |
| ✅ 树模型可省略标准化                             | 但保留也无害                             |
| ✅ 高基数类别不用 One-Hot                         | 改用 Target Encoding、Embedding、Hashing |
