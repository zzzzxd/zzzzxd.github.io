### 题目解析

- 题目的意思两条路径不能走相同的，这个题可以变通一下，变成同时走两条路，只不过走相同的位置只能获取一次，而且任意时刻，走的总格子数肯定是一样的
- 用递归是没问题的，但是必须优化，直接写递归可能是4个参数，开辟的缓存太大。我们发现，不管怎么走，任意时刻两个人走的总步数是一致的。可以根据这，降低一个维度。

### 代码

```java
public static void main(String[] args) {
        //仔细分析之后吗，先去一次，再回来一次，等价于去了一次，在这状态上再去一次，只不是拿了之后就不能再拿了，那么，相当于两个人一块去，同时一个地方的时候，
        //只让一个人拿

        final Scanner scanner = new Scanner(System.in);
        final int M = scanner.nextInt();
        final int N = scanner.nextInt();
        final int[][] ints = new int[M][N];
        for(int i = 0;i < M;i++){
            for(int j = 0;j < N;j++){
                  ints[i][j] = scanner.nextInt();
            }
        }
//        final HashMap<String, Integer> dp = new HashMap<>(); hashmap依然超时
//        int[][][][] dp = new int[M][N][M][N]; //挂混存这个太大了，最后一个case 堆溢出了 50多M
        //可是直接做dp 四个变量，四个纬度，边界都搞不出来来，比较麻烦
        //所以还是想办法降低缓存大小,这个人的位置是二维，降成一维
        int[][][] dp = new int[M][N][M];
        for (int i = 0;i < M;i++){
            for (int j = 0;j < N ;j++){
                for(int k = 0;k < M;k++) dp[i][j][k] = -1;
            }
        }
        System.out.println(process(0, 0, 0, ints, M, N,dp));
    }
    // 定义函数为 这两人到达右下角获取的的最大价值
    public static int process(int i1,int j1,int i2,int[][] map,int M,int N, int[][][] dp){
        if(i1 >= M || j1 >= N || i2 >= M || (i1+j1-i2) >= N) return 0; //越界

        if(dp[i1][j1][i2] != -1) return dp[i1][j1][i2];
        if(i1 == M-1 && j1 == N-1){ //到达
            dp[i1][j1][i2] = map[i1][j1];
            return map[i1][j1];
        }
        int p1 = process(i1,j1+1,i2,map,M,N,dp);
        int p2 = process(i1,j1+1,i2+1,map,M,N,dp);
        int p3 = process(i1+1,j1,i2,map,M,N,dp);
        int p4 = process(i1+1,j1,i2+1,map,M,N,dp);
        final int max = Math.max(Math.max(Math.max(p1, p2), p3), p4);
        int res = max + (i1 == i2 && j1 == (i1+j1-i2) ? map[i1][j1] : map[i1][j1] + map[i2][i1+j1-i2]);
        dp[i1][j1][i2] = res;
        return res;
    }
```
