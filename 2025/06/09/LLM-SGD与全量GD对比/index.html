✅ SGD vs 全量梯度下降：为什么深度学习都用“小批量”？

🎯 核心结论（先看答案）

特性 全量梯度下降（Batch GD） 随机梯度下降（SGD / Mini-batch SGD）
使用数据 整个训练集 一个样本 或 一个小批量（如 32、64）
梯度计算 精确，方向稳定 有噪声，方向波动
更新频率 每遍历完整个数据集才更新一次 每个 batch 更新一次
训练速度 极慢（内存、计算压力大） 快得多
收敛行为 平滑，但易陷局部最优 有噪声，可能跳出坏的局部最优
实际使用 ❌ 几乎不用 ✅ 深度学习标准做法

✅ 现代深度学习几乎都用 SGD 或其变体（如 Adam），根本原因：效率 + 泛化 + 可扩展性

🔍 一、全量梯度下降（Batch Gradient Descent）

是什么？

  • 每次更新参数前，计算整个训练集的平均梯度。

  • 公式：

优点

  • 梯度方向非常准确
  • 更新路径平滑，收敛稳定

缺点（致命！）

  1. 太慢：每更新一步都要遍历全部数据（如 100 万张图）
  2. 内存爆炸：无法将全部数据加载到 GPU
  3. 容易陷入局部最优：没有噪声帮助跳出
  4. 无法在线学习：必须等所有数据齐备

📌 适合：数据量极小（几百条）、凸优化问题（如线性回归）

🔍 二、随机梯度下降（SGD）与小批量 SGD（Mini-batch SGD）

是什么？

  • SGD：每次只用一个样本计算梯度并更新
  • Mini-batch SGD：每次用一小批样本（如 32、64、128）计算平均梯度

这是现代深度学习的标准做法。

优点

  1. ✅ 速度快：每 batch 更新一次，1 个 epoch 就能更新很多次
  2. ✅ 内存友好：只需加载小 batch 到 GPU
  3. ✅ 利用硬件并行：GPU 擅长并行计算小矩阵
  4. ✅ 噪声有助于泛化：梯度有波动，可能跳出尖锐的局部最优，找到更平坦的极小值（泛化更好）
  5. ✅ 支持在线学习：数据可以持续流入

缺点

  • 梯度有噪声，更新路径震荡
  • 学习率需要 careful 调整

🧠 三、为什么“噪声”反而是好事？

听起来反直觉：不准确的梯度怎么反而更好？关键在于：

深度学习的损失函数极其复杂，有很多“尖锐的局部最优”和“平坦的全局最优”。

  • 全量 GD：像一个“盲人拄拐”，走得很稳，但一旦掉进一个尖锐的小坑，就出不来了。
  • SGD：像一个“带震动的球”，虽然走不直，但震动（噪声）可能把它从坏的小坑里“抖”出来，最终滚到更平坦、泛化更好的区域。

📌 这就是 SGD 的隐式正则化效应：噪声 → 更好泛化。

🔄 四、实际训练流程（以 Mini-batch SGD 为例）

  • 一个 epoch：遍历完整个数据集
  • 但参数更新了 N / batch_size 次！

🆚 五、SGD vs 全量 GD：直观对比

场景 全量 GD Mini-batch SGD
数据量 100 万 更新一次要几小时 每秒可更新几十次
GPU 显存 必须存下全部数据 → 不可能 只存一个 batch → 轻松
收敛曲线 平滑下降 上下震荡，但总体下降
泛化性能 可能过拟合 通常更好
实际使用 ❌ 基本不用 ✅ 标准做法

✅ 六、总结：为什么深度学习都用 SGD？

  1. 效率：小批量更新快，适合大规模数据
  2. 可扩展：能用 GPU 并行，支持大数据
  3. 泛化好：梯度噪声有正则化作用
  4. 工程友好：内存可控，支持流式数据
  5. 基础框架：Adam、RMSProp 等都是 SGD 的改进版

✅ 所以：不是“为什么用 SGD”，而是“为什么不用全量 GD”。
