🔹 题1：【快手｜2024秋招终面】

场景：用户连续登录天数统计（容错版）
规则：

  • 若用户在日期序列中最多只中断1天（即：存在某日未登录，但前后都是登录日），仍视为“连续”；
  • 中断≥2天（如：登-登-不登-不登-登）则断裂；
  • 要求：找出每个用户的最长连续登录段天数（按自然日计算，非行为次数）。

输入表：dwd_user_login_log

| —————— | ——— | ———————————————————- |
| uid | STRING | 用户ID |
| login_date | STRING | 登录日期，格式 '2026-03-01'（无时间） |
| dt | STRING | 分区字段（同 login_date） |

🔹 题5：【拼多多｜2024.11 笔试】

场景：用户“会话内连续点击商品”路径长度
规则：

  • 先按用户+会话分组（会话定义：30分钟无行为）；
  • 在每个会话内，统计连续点击不同商品ID的最长链长度（即：item_id 序列中，相邻两项不同时的最大子序列长度）；
  • ⚠️ 注意：不是“点击次数”，而是“切换商品”的次数 + 1（如：A→B→C→C→D，连续不同为 A→B→C→D，长4）。

| uid | STRING | 用户ID |
| session_id | STRING | 会话ID（已生成） |
| item_id | STRING | 商品ID |
| click_time | STRING | 点击时间 |
