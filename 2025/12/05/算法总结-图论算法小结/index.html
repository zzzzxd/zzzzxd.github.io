🗺️ 图论刷题全景地图 (熟练度提升指南)

图论题目看似千变万化，其实核心套路只有 3 种遍历/搜索方式 和 2 种高级算法。掌握以下模板，80% 的中等题都能秒杀。

1. 基础数据结构 (必须烂熟于心)

场景 推荐结构 代码示例 (Java) 适用情况
稀疏图 (边少点多) 邻接表 (最常用) List<List<Integer>> adj = new ArrayList<>(); 或 ArrayList<Integer>[] adj = new ArrayList[n]; 绝大多数算法题 (BFS, DFS, 拓扑)
稠密图 (边多点少) 邻接矩阵 int[][] grid = new int[n][n]; 网格题 (岛屿问题), 节点数 < 200
带权图 邻接表 + Pair List<List<int[]>> adj int[]{neighbor, weight} 最短路径 (Dijkstra)

2. 三大核心算法模板

A. 广度优先搜索 (BFS) —— 求“最短距离”、“层级”
  • 核心思想：层层推进，第一次遇到目标时的步数就是最短。
  • 数据结构：Queue + visited 数组/集合。
  • 典型题目：
    • 单词接龙 (LeetCode 127)
    • 打开转盘锁 (LeetCode 752)
    • 地图中的最高点 (LeetCode 1765)
    • 拓扑排序 ：本质也是 BFS，只不过入队条件是 inDegree == 0。
B. 深度优先搜索 (DFS) —— 求“连通性”、“所有路径”、“环检测”
  • 核心思想：一条路走到黑，撞墙回溯。
  • 数据结构：递归栈 (隐式) 或 Stack (显式) + visited。
  • 典型题目：
    • 岛屿数量 (LeetCode 200)
    • 被围绕的区域 (LeetCode 130)
    • 课程表 II (输出顺序)
    • 环检测 (进阶)：需要 onPath 数组辅助判断当前递归栈中是否有环。
C. 并查集 (Union-Find) —— 求“动态连通性”、“连通分量个数”
  • 核心思想：把相连的节点归到一个“老大”底下。
  • 数据结构：parent 数组。
  • 典型题目：
    • 冗余连接 (LeetCode 684)
    • 省份数量 (LeetCode 547)
    • 满足交换条件的最大元素 (LeetCode 1202)
  • 优势：比 BFS/DFS 处理动态连接更高效，代码极短。

3. 高级算法 (针对带权图)

  • Dijkstra 算法：求单源最短路径（权重非负）。

    • 数据结构：PriorityQueue (最小堆) + dist 数组。
    • 题目：网络延迟时间 (LeetCode 743), 最小体力消耗路径 (LeetCode 1631)。
    • 关键点：每次从堆里取出距离最小的点去更新邻居。

  • 拓扑排序 (Topological Sort)：求依赖顺序、判环。

    • 方法一 (Kahn)：BFS + 入度表 (你做的这个，适合求具体顺序)。
    • 方法二 (DFS)：后序遍历反转 + 判环 (适合只判环或需要递归场景)。

⚡ 如何提升做题速度？(训练计划)

  1. 肌肉记忆建图：

    • 看到 int[][] edges 或 List<List<Integer>> graph，要在 30 秒内写出邻接表的初始化代码。
    • 分清是有向图还是无向图（无向图要加两次边）。

  2. 识别题型关键词：

    • “最少次数”、“最短路径” $\rightarrow$ BFS。
    • “能否到达”、“所有可能”、“岛屿” $\rightarrow$ DFS。
    • “依赖关系”、“先后顺序”、“有没有环” $\rightarrow$ 拓扑排序。
    • “连通分量”、“分组”、“是否连通” $\rightarrow$ 并查集。
    • “最小代价”、“权重” $\rightarrow$ Dijkstra (或 BFS 如果权重为 1)。

  3. 固定变量命名：

    • 别每次都想变量名。统一用：
      • adj / graph: 邻接表
      • visited: 访问标记
      • inDegree: 入度
      • dist: 距离
      • pq: 优先队列
    • 这样你的大脑可以专注于逻辑，而不是语法。

  4. 专项突破：

    • 这周只刷 BFS 相关的图论题 (5-10 道)。
    • 下周只刷 DFS 相关的。
    • 再下周刷 并查集。
    • 集中训练能帮你快速建立模式识别能力。

📝 总结

你现在的水平已经能够独立解题，接下来的目标是“标准化”和“自动化”。
把上面的 BFS/DFS/拓扑/并查集 四个模板背下来（理解着背），下次做题时，先花 1 分钟判断类型，然后直接套用模板填充逻辑。这样你的速度至少能提升一倍！
