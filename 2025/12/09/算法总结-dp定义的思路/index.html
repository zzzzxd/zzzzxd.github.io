“定义成和历史状态无关的”，正是动态规划（DP）和记忆化搜索的灵魂所在。

在算法圈里，这被称为 “最优子结构” (Optimal Substructure) 的核心体现。

🧠 为什么必须“和历史无关”？

1. 递归的本质是“解决子问题”

当你调用 dfs(index) 时，你是在问：

“嘿，假设我现在站在 index 这个位置，不管我之前是怎么走到这里的，从现在开始，我能拿到的最大利益是多少？”

如果这个答案依赖于“之前走了哪条路”或者“之前攒了多少钱”，那么这就不是一个独立的子问题，而是一个特定路径的历史记录。

  • 独立子问题：只要输入参数（状态）一样，输出结果就一定一样。（可以缓存/记忆化）
  • 依赖历史：即使参数一样，因为“前世今生”不同，结果也不同。（无法缓存，只能暴力搜）

2. 状态压缩的关键

只有当返回值与历史无关时，我们才能把无数个不同的“历史路径”合并到同一个“当前状态”上。

  • 错误定义：dfs(index, currentSum)
    • 路径 A: 0->2->4 (Sum=10) 到达 index=5
    • 路径 B: 1->3->4 (Sum=12) 到达 index=5
    • 系统认为这是两个不同的状态，需要分别计算后续。缓存失效！
  • 正确定义：dfs(index) 返回 futureMax
    • 路径 A 到达 index=5 -> 询问未来最大值 -> 得到 X -> 总结果 = 10 + X
    • 路径 B 到达 index=5 -> 询问未来最大值 -> 得到 X (直接读缓存!) -> 总结果 = 12 + X
    • 缓存命中！ 无论你怎么来的，从 5 号位往后看的风景是一样的。

🛠️ 如何训练这种思维？（通用公式）

以后遇到任何 DP 或 记忆化搜索题目，按照这个模板来定义函数：

❌ 错误的思维模式（过程导向）

“我要写一个函数，它负责走完剩下的路，并把最终的总金额带回来。”

  • 参数往往包含：index, currentSum, pathList…
  • 结果：状态爆炸，超时。

✅ 正确的思维模式（状态导向）

“我要写一个函数，给定当前的状态 S，告诉我从这一刻起的最优解是多少。”

  • 步骤 1：找变量。哪些因素会影响未来的决策？
    • 打家劫舍：当前位置 i，前一个是否偷了 flag。
    • 背包问题：当前物品索引 i，剩余容量 w。
    • 股票买卖：当前天数 i，是否持有股票 hold，交易次数 k。
  • 步骤 2：砍掉累加器。绝对不要把 currentProfit, currentSum, currentPath 放进参数！
  • 步骤 3：定义返回值。返回值 = 从当前状态出发能获得的增量最大值。

🌰 再举两个例子巩固一下

案例 1：背包问题 (Knapsack)

  • 题目：背包容量 W，物品有重量 w[i] 和价值 v[i]，求最大价值。
  • ❌ 错误定义：
    dfs(index, currentWeight, currentValue)
    • 原因：currentValue 是历史积累，导致状态数爆炸。
  • ✅ 正确定义：
    dfs(index, remainingCapacity)
    • 含义：从第 index 个物品开始，在剩余容量为 remainingCapacity 的情况下，往后能装的最大价值。
    • 转移方程：max(不装, 装 + dfs(index+1, remainingCapacity - w[i]))
    • 最后结果：dfs(0, W)

案例 2：最长递增子序列 (LIS)

  • 题目：找数组中最长递增子序列的长度。
  • ❌ 错误定义：
    dfs(index, currentSequenceList)
    • 原因：记录具体的序列内容，无法记忆化。
  • ✅ 正确定义：
    dfs(index) （通常需要配合外层循环，或者定义为以 index 结尾的最长长度）
    • 更常见的 DP 定义：dp[i] = 以第 i 个元素结尾的最长递增子序列长度。
    • 这里的状态只和 i 有关，和前面具体选了哪些数无关（只要知道前面的最大值即可）。

🚀 总结口诀

写递归/DP 函数签名时，默念三遍：

  1. 参数只看“未来”：只放影响后续决策的变量（索引、剩余量、开关状态）。
  2. 坚决不要“过去”：严禁放入累加和、路径列表、计数器等历史产物。
  3. 返回值是“增量”：函数返回的是“从现在开始能赚多少”，而不是“一共赚了多少”。

只要做到这三点，你的记忆化搜索一定能跑满效率，状态数永远控制在 $O(N^k)$ 的多项式级别，而不是指数级！

你现在对 DP 的理解已经上了一个大台阶，接下来可以尝试去刷一下 LeetCode 309 (最佳买卖股票时机含冷冻期) 或 LeetCode 714 (买卖股票的最佳时机含手续费)，用这个新思维去定义状态，会发现特别顺手！
