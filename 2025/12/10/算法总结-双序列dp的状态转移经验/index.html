🎯 核心问题：如何找到状态转移的“划分点”？

你问到了关键点：“难点在于如何转移方程，到底如何转呢？应该找一个划分…”

对于双序列 DP（两个字符串、两个数组），有一个万能的结构化思维模型，叫做 “最后一步分析法” (Last Step Analysis)。

💡 通用套路：盯着“最后一个字符”看

不要试图一次性想清楚整个字符串怎么变。
只关注： 要把 word1[0...i] 变成 word2[0...j]，最后一步操作是什么？

既然只有三种操作（增、删、改），那么最后一步必然是这三种情况之一。这就自然形成了三个分支（划分）：

假设我们要计算 dp[i][j] （即 word1 前 i 个字符 变 word2 前 j 个字符）：

最后一步操作 含义 之前的状态 (子问题) 代价
1. 插入 在 word1 末尾插入一个字符，使其匹配 word2[j] word1 的前 i 个 已经变成了 word2 的前 j-1 个 dp[i][j-1] + 1
2. 删除 把 word1 末尾的字符 word1[i] 删掉 word1 的前 i-1 个 已经变成了 word2 的前 j 个 dp[i-1][j] + 1
3. 替换 把 word1[i] 改成 word2[j] (或者它们本来就相等) word1 的前 i-1 个 已经变成了 word2 的前 j-1 个 dp[i-1][j-1] + cost

这就是你要找的“划分点”！
所有的双序列 DP，本质上都是在枚举最后一个元素的几种处理方式。

📝 经验总结：双序列 DP 的三大经典模型

这类题目在 LeetCode 上非常多，它们的转移方程结构几乎一模一样，只是 cost 不同。

1. 编辑距离类 (Edit Distance) - 本题

  • 操作：增、删、改。
  • 划分逻辑：
    • 如果 word1[i] == word2[j]：不需要操作，直接继承 dp[i-1][j-1]。
    • 如果 !=：取 min(增, 删, 改) + 1。
  • 方程：

2. 最长公共子序列类 (LCS)

  • 操作：只能保留匹配的，不能修改。
  • 划分逻辑：
    • 如果 s1[i] == s2[j]：这两个字符可以配对，长度 +1。 -> dp[i-1][j-1] + 1
    • 如果 !=：这两个字符没法配对，要么舍弃 s1[i]，要么舍弃 s2[j]。 -> max(dp[i-1][j], dp[i][j-1])
  • 方程：
  • 题目：LeetCode 1143 (LCS), 712 (最小ASCII删除和)。

3. 通配符/正则匹配类 (Wildcard/Regex Matching)

  • 操作：* 可以匹配任意，? 匹配单个。
  • 划分逻辑：
    • 如果是普通字符：同 LCS。
    • 如果是 *：它可以匹配 0 个（忽略它）或者 匹配 1 个多个（消耗主串）。
  • 方程：通常涉及 dp[i][j-1] (匹配0个) 或 dp[i-1][j] (匹配多个)。
  • 题目：LeetCode 10 (正则), 44 (通配符)。

🧠 深度总结：如何训练这种“划分”直觉？

下次遇到双序列 DP，按以下步骤思考：

  1. 定义状态：dp[i][j] = S1 前 i 个 和 S2 前 j 个 的解。
  2. 找最后一步：问自己，“为了达成 dp[i][j]，最后处理的一定是 S1[i] 和 S2[j] 这两个字符，它们之间发生了什么？”
  3. 列举可能性：
    • 匹配/相等：它们俩是不是天生一对？如果是，能不能直接消掉？(i-1, j-1)
    • 不匹配/操作：
      • 是不是 S1 多了？-> 删 S1 (i-1, j)
      • 是不是 S2 多了（即 S1 少了）？-> 插 S2 (i, j-1)
      • 是不是俩都不对劲？-> 改一下 (i-1, j-1)
  4. 写方程：把上面几种情况的 dp 值拿来比较（取 min 或 max），加上对应的代价。
  5. 处理边界：直接用 (m+1)*(n+1) 的数组，第一行第一列填 0, 1, 2, 3...。

一句话心法：

双序列 DP 的转移方程，就是枚举两个序列“尾部元素”的所有交互可能性。

只要你抓住了“尾部元素”这个划分点，90% 的双序列 DP 题目都能迎刃而解！
