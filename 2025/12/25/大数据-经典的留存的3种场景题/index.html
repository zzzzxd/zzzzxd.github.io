📊 题目一：经典的“次日留存率” (基础必考)

来源：字节跳动 / 美团
难度：⭐⭐
核心考点：自连接 (Self-Join) 或 窗口函数 LEAD，去重逻辑。

1. 题目描述

有一张用户登录表 user_login (user_id, login_date)。
请计算每一天的次日留存率。

  • 定义：某日的新增用户中，在第二天也登录了的用户比例。
  • 注意：这里通常指“该日首次登录的用户”（即新增用户），而不是“该日所有活跃用户”。如果题目没说“新增”，默认按“当日活跃用户”算，但面试时最好问清楚。我们这里按新增用户（更严谨）来写。

2. 解题思路

  1. 找出每日新增用户：每个用户的第一次登录日期。
  2. 关联次日数据：看这些新用户是否在 login_date + 1 有记录。
  3. 聚合计算：Count(次日登录) / Count(总新增)。

3. Hive SQL 代码

💡 面试官追问：如果数据量很大，LEFT JOIN 会不会慢？
回答：如果表有分区且 user_id 分布均匀，Shuffle 开销可控。另一种优化是用窗口函数 LEAD 拿到下一次登录时间，直接判断 DATEDIFF(next_login, first_login) = 1，这样可以避免大表自连接，只需一次扫描。

📊 题目二：N日留存率矩阵 (进阶通用)

来源：阿里巴巴 / 腾讯
难度：⭐⭐⭐
核心考点：条件聚合、动态列转行（或行转列）、多日窗口判断。

1. 题目描述

同上表 user_login。
请计算每日新增用户的 1日、3日、7日 留存率，并输出在一行中。

  • 输出格式：date, new_users, day1_retention, day3_retention, day7_retention。
  • 定义：
    • 1日留存：新增后第1天（即次日）活跃。
    • 3日留存：新增后第3天（注意：是第3天，不是3天内）活跃。
    • 7日留存：新增后第7天活跃。

2. 解题思路

  1. 先确定每个用户的新增日期 (first_dt)。
  2. 计算该用户每次登录日期与新增日期的差值 (diff)。
  3. 利用 SUM(CASE WHEN diff = N THEN 1 ELSE 0 END) 进行条件聚合，一次性算出多个留存指标。

3. Hive SQL 代码

💡 关键点：

  • 这里用 JOIN 而不是 LEFT JOIN 是因为 login_diff CTE 里我们只关心那些确实发生了登录的行为。如果用户第3天没来，diff_days=3 的记录就不存在，SUM 自然为 0，逻辑正确。
  • 区分“第N天”和“N天内”：题目如果是“3日内留存”，条件改为 diff_days BETWEEN 1 AND 3。

📊 题目三：连续活跃留存 (高难度变种)

来源：抖音 / 快手 (直播/短视频业务)
难度：⭐⭐⭐⭐
核心考点：结合“连续登录”逻辑 + “留存”逻辑。考察对业务场景的深度理解。

1. 题目描述

不仅要算留存，还要算“高质量留存”。
计算每日新增用户中，在随后 7 天内，至少有 3 天连续活跃的用户占比。

  • 场景：很多用户可能第1天来，第7天来，中间都不来。虽然满足“7日留存”的定义（第7天来了），但粘性很差。业务方想看的是“是否形成了使用习惯”（即中间有过连续3天及以上）。

2. 解题思路

这是一个组合拳：

  1. 确定新增日 (first_dt)。
  2. 筛选窗口：只看该用户 first_dt 到 first_dt + 6 (共7天) 的登录记录。
  3. 判断连续性：在这7天窗口内，是否存在一段长度 >= 3 的连续登录。
    • 这里可以复用我们之前讲的 date - row_number 或者 LAG 打标法。
  4. 聚合计算：统计满足条件的用户数 / 总新增用户数。

3. Hive SQL 代码 (逻辑拆解版)

💡 杀手锏回答：
面试官如果问性能，你可以说：“这个查询涉及多次 Shuffle。在离线数仓中，我们可以将 user_login 表按 user_id 分桶，或者先将数据预聚合为‘用户-日期’宽表。如果在实时场景（Flink），我会用 KeyedProcessFunction 维护一个滑动窗口状态，当用户登录时更新连续计数，一旦达到3天立即打标，这样无需回溯历史数据。”

📝 总结：留存问题的万能模板

无论题目怎么变（次日、7日、N日连续），核心步骤永远是这三步：

  1. 定基准 (Anchor)：
    • 是“每日活跃”做基准？还是“每日新增”做基准？
    • SQL: MIN(date) OVER (...) 或 GROUP BY date。
  2. 定窗口 (Window)：
    • 是“精确第N天”？还是“N天内任意一天”？还是“连续N天”？
    • SQL: DATEDIFF 过滤，或 BETWEEN，或 ROW_NUMBER 分组。
  3. 算比率 (Aggregate)：
    • SQL: SUM(CASE WHEN ... THEN 1 ELSE 0 END) * 1.0 / COUNT(*)。

掌握这三个模板，90% 的留存类 SQL 题都能迎刃而解！需要我针对某一个点再深入展开吗？
