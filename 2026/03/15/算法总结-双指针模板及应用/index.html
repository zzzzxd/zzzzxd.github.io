🧩 通用模板回顾（所有变种的基础）

关键：数组必须有序，条件具有单调性（l 增大 → 合法 r 不减小）

🔁 变种 1：选 4 个点，最大最小差 ≤ D

📌 题目描述

给定 n 个已排序的建筑坐标和距离上限 D，
求有多少种选择 4 个不同建筑 的方案，使得最远两建筑距离 ≤ D。

示例：[1,2,3,4,5], D=3 → 合法四元组如 (1,2,3,4)（4-1=3≤3）

🔍 分析

  • 固定左端点 l
  • 找最大 r 使得 a[r] - a[l] <= D
  • 中间有 cnt = r - l - 1 个点
  • 需从中选 3 个（因为左端点已定）
  • 方案数 = C(cnt, 3) = cnt*(cnt-1)*(cnt-2)/6

✅ 代码实现

🔁 变种 2：正数数组中，子数组和 ≤ S 的子数组个数

📌 题目描述

给定一个正整数数组 nums 和上限 S，
求有多少个连续子数组满足 sum(subarray) <= S。

示例：nums=[1,2,3], S=3 → [1],[2],[3],[1,2] → 答案=4

🔍 分析

  • 因为是正数数组，前缀和单调递增
  • 固定左端点 l，找最大 r 使得 sum(l..r-1) <= S
  • 子数组个数 = r - l（从 l 开始，长度 1 到 r-l 都合法）

💡 注意：这里不是组合数，而是连续子数组计数

✅ 代码实现

⚠️ 这里维护的是窗口 [l, r) 的和，所以用 sum 变量

🔁 变种 3：字符串中最短覆盖子串（含所有字符）

📌 题目描述

给定字符串 s 和 t，找到 s 中最短子串，使其包含 t 中所有字符（包括重复）。

示例：s="ADOBECODEBANC", t="ABC" → "BANC"

🔍 分析

  • 使用滑动窗口 + 哈希表
  • 扩展右指针直到窗口包含 t
  • 收缩左指针直到不包含，记录最小长度

💡 虽然不是组合计数，但同样是“同向双指针”模板

📊 三种变种对比总结

变种 数组性质 条件 计数方式 是否需要组合数
选 k 个点 有序 最大-最小 ≤ D C(cnt, k-1) ✅ 是
子数组和 正数 和 ≤ S r - l ❌ 否
最短覆盖 字符串 包含所有字符 记录最小长度 ❌ 否

✅ 核心思想统一

所有“同向双指针”问题的本质：利用单调性，避免重复计算。

  • 当左指针右移时，右指针不需要回退
  • 每个元素最多被访问两次（O(n)）
  • 关键是找到正确的窗口收缩/扩展条件
